File Upload
(Vulnerable to Unvalidated Upload)

Vulnerability: Unvalidated File Upload

What is it? Unvalidated file upload occurs when a web application allows users to upload files without checking their type, content, or size. Attackers can upload malicious files (e.g., scripts, web shells) that can be executed on the server or delivered to other users.

How does it work here? This upload form accepts any file and saves it to the server without validation. An attacker could upload a script or executable file.

Example Attack:
Upload a file named malicious.php or shell.js. If the server executes or serves this file, it could be used to compromise the system.

Mitigation Strategies:
  • Validate file type, size, and content on the server side.
  • Rename files and store them outside the web root.
  • Set correct permissions and never execute uploaded files.
  • Use antivirus/malware scanning on uploads.
